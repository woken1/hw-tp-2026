# merge2 - Слияние двух отсортированных массивов

## Что делает функция

Объединяет два отсортированных массива в один отсортированный массив.

## Сигнатура

template< class T, class Compare = default_compare<T> >
T* merge(const T* a, size_t sa, const T* b, size_t sb, T* c, size_t sc, Compare comp = Compare());

## Неявный интерфейс

### Требования к типу T

Тип T должен поддерживать:
1. Копирование — T t1 = t2;
2. Присваивание — t1 = t2;
3. Сравнение — a < b (если не указан свой компаратор)

### Требования к компаратору

Если используете свой компаратор, он должен иметь вид:
bool operator()(const T& a, const T& b)

Возвращает true, если a < b.

Может быть: лямбда-выражение, функтор (класс с operator()) или обычная функция.

## Как это работает

Код:
int a[] = {1, 2, 3};
int b[] = {2, 4, 5};
int c[10];
int* end = merge(a, 3, b, 3, c, 10);

Пояснение:
- a и b — входные отсортированные массивы
- c — выходной буфер
- После выполнения в c будет: 1 2 2 3 4 5
- end указывает на позицию после последнего элемента

## Безопасность

Проверки:
- Проверяет null-указатели
- Проверяет размер выходного буфера
- Бросает исключения при ошибках

Гарантии:
Если возникнет исключение — массив c останется валидным (частично заполненным).

## Пример с компаратором

Код:
auto comp = [](int x, int y) { return x > y; };
merge(a, 3, b, 3, c, 10, comp);

Пояснение:
- Компаратор задаёт порядок убывания
- Результат: 6 5 4 3 2 1

## Исключения

my_exception — базовый класс для всех исключений

invalid_argument — некорректные аргументы (null-указатели)

length_error — недостаточно места в выходном буфере

## Сборка

g++ -std=c++11 -o merge main.cpp
./merge

## Зависимости

Только стандартная библиотека C++11. Никаких внешних зависимостей.
